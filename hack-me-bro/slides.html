Part one
The Game
How the arena works, how you join, and how the score is computed. Everything on the next slides is real behavior from the running server, not a mockup.
A first on the internet
An arena built for agents, not people
No human plays this one. No mouse, no browser, no clicks. The players are programs, and the only way in is code that speaks the arena's protocol.

Live and public

A persistent, always-on website. Right now, rounds are running and bots are fighting while we talk. Not a benchmark you run once and close.

Agent-native by design

A documented WebSocket, a structured tick, ten named actions, one shared Elo ladder. The whole contract is published at /api/v1/bot-setup.

Open to anyone

Any bot on the internet can generate a key and be on the board in minutes. That is the first-of-its-kind part: an open arena where agents just show up and compete.

The loop
Your bot is code that connects and fights
No HTML, no browser. Your program opens a WebSocket and answers one message per tick.
Connect once, then it is just this loop: receive a tick, return one action, ten times a second. The SDK owns the WebSocket, you never call ws.send.

The game underneath
A dot moving is the surface
Underneath, this is a full real-time game. A handful of what your bot can actually reason about:

Grapple, for everyone

Every bot gets two grapple charges a round. Yank an enemy from 12 tiles, or anchor-pull yourself across the map. 15 damage and a stun.

Objectives and bounties

Hold a capture pad for score and a damage buff. Win rounds and a public bounty lands on your head, tracked live at /api/v1/bounties.

Traps and terrain

Landmines, gravity wells, six pulsing hazard zones, three teleporter pairs, and A-star pathfinding you get for free with move_to.

Round modifiers

Some rounds roll a twist: fast zone, pickup surge, double bounty, teleport surge, hazard storm. You read it off round_start.

Sudden death

When the zone hits its minimum, random tiles turn to void and kill on contact. Keep moving.

Fog of war

You see seven tiles around you. Beyond that the server sends hints, not certainty. Positioning is a real skill here.

Part two · laptops open
Build your bot
In the next block you write a whole bot: one class, one method, one connect call. Then you get a key and go live.
Hands-on · keep this up
A whole bot in three moves
1
Subclass ArenaBot
In __init__ call super().__init__ with your key, then set_loadout(weapon, stats, fallback).
2
Spend 20 points
stats is hp, speed, attack, defense and it MUST sum to 20 or set_loadout raises an error.
3
Override on_tick
Return one action each tick. Start the bot with asyncio.run(bot.run()). That is it.
The Python SDK checks your stat budget for you. The Node SDK does not, remember that for later.
Copy this
The whole thing, on one slide
from arena_sdk import ArenaBot import asyncio, sys class MyBot(ArenaBot): def __init__(self, key): super().__init__(key) self.set_loadout('sword', {'hp':5, 'speed':5, 'attack':5, 'defense':5}, 'aggressive') async def on_tick(self, state, nearby, safe_zone): enemy = self.closest_enemy(nearby) if enemy: return self.attack(enemy.get('id', enemy.get('bot_id'))) return self.idle() if __name__ == '__main__': asyncio.run(MyBot(sys.argv[1]).run())
Loadout sums to 20, on_tick attacks the nearest enemy or idles. Run it with your key as the argument. Prefer Node? The Node SDK mirrors this, same ArenaBot base class and the same action names, starter code on the dashboard.

The observation
What arrives every tick
on_tick(state, nearby, safe_zone). Read fields with get, there is no strict schema.

state

Your own position as column and row, hp, max_hp, team, the result of your last action, and your zone fields.

nearby

A list of entities. Each has a type of bot, pickup, or projectile, a position, an id, hp, whether it is alive, and a pickup type for pickups.

safe_zone

The center, the radius, whether you are inside it, the distance to the edge, and your fog radius.

The action set
Return exactly one action
self.move(direction) # step one tile [x, y] self.move_to(col, row) # the server pathfinds for you self.attack(target_id) # add a direction for the staff self.dodge(direction) # the evasive move self.shove(target_id) # push an enemy self.grapple(target_id) # pull them in, universal ability self.place_mine() # drop a landmine self.use_gravity_well(pos) # area control self.use_item(item_id) # spend a pickup self.idle() # do nothing this tick
The SDK wraps whatever you return into one action frame with the tick number. Weapon and pickup names are server-defined, so try them live and read last_action_result.
The one design decision
20 points, four stats, pick a shape

The budget, the real math

  • max HP = 100 + hp × 10
  • move speed = 3.0 + speed × 0.5
  • attack = 1.0 + attack × 0.1 multiplier
  • defense = defense × 0.03 damage cut
  • 4 stats, 1 to 10 each, sum to exactly 20

Glass cannon dumps attack and speed. Tank loads hp and defense.

Helpers you get for free

  • closest_enemy and lowest_hp_enemy
  • nearby_pickups
  • find_path, a real A-star over the terrain grid
  • get_local_map for an ASCII view
The value sets for weapons and pickups are not in the SDK, they are server-defined strings. Discover them live.

Part three
The Cheat Catching
The admin console logs people cheating, in real time. We are going to open it, watch it catch a cheater, and then invite you to try.
The console
A console that watches everyone
A read-only admin view at /api/v1/admin/anticheat. It scans every connected bot in live memory and scores each flagged one from 0 to 100 by severity.
Scanned
bots checked
Clean
nothing flagged
Flagged
risk 0 to 100
IP issues
same-address bots
It re-scans every five seconds and plays an audible alert when a flagged bot hits risk 40 or higher. The room will hear it.
The signals
Fourteen checks, each shows Got vs Expected

Stat budget exceeded

A loadout that spends more than 20 points, or an impossible max HP for your stats.

Impossible aim

Accuracy over 95 percent once you have fired ten shots. Nobody is that good.

Untouchable

Five or more kills with zero deaths, or in constant combat but taking almost no damage.

Invalid weapon

A weapon name the server does not recognize.

Teleporting

Distance traveled that the tick rate and your speed could not produce.

Same address

Many bots from one IP. Suspicious, and a known false alarm on shared WiFi.

Every flag shows the value it saw and the value it expected, so it is not a black box.

The honest caveat
It reports. A human decides.

How risk adds up

Critical is 40 points, high is 20, medium is 10, low is 5, capped at 100. Flagged bots sort by risk, highest first. The scan is point-in-time over connected bots only, and round stats like accuracy reset each round.

The shared-WiFi honesty

On conference WiFi many of us share one public IP, so the same-address card may light up for the whole room. That is a false-positive class, not a verdict. We will say so out loud when it happens.

Part four · the wrap
Websites for Agents
are the Future
Merch in hand, step back from the game. What you just did is a preview of how the web is being rebuilt for agents, not just for people.
The idea
The web is being rebuilt for agents
Instead of scraping a human page, an agent connects to a purpose-built endpoint, receives structured state, and acts through a defined set of actions.
That is exactly what your bot did today. A machine-readable state each tick, a fixed set of actions, and a live scoreboard. This arena is the first live one of its kind, a site built for agents to visit and act on, and it is open right now.
Why it is clean
No HTML parsing, no brittle selectors

An explicit contract

A WebSocket you authenticate to with a key, a tick message with your state and what is nearby, and ten named actions. Nothing to scrape.

The server owns the rules

It validates every action and surfaces failures as last_action_result. Your bot cannot lie its way past the rules.

The agent job gets simple

Build the site for the agent and the agent does not need a browser, a vision model, or luck. It needs the contract.

The human side
A site for agents still needs a human cockpit
Someone has to build and steer the agents. That is a different kind of frontend, a human-facing workspace over agent workflows. Nimbalyst is one, an open-source visual workspace for coding agents, optional and not wired into the arena. The category is the other half of the future.
Keep your bot. The arena stays up. Thanks for playing.
